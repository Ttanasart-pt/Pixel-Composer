MK Tile generate tileset from a set of surfaces.

## Layers

Tile set generation has 3 layers, the background <junc background texture/>, tile <junc texture/>, and edges 
<junc edge/>. The surface provided to every layers should be tilable/seamless texture.

<img mk_tile_layers/>

## Tile Set

The <junc type/> control the set of tiles to be generated. Currently there're 3 sets:

### GMS Corner

Tile set for GameMaker studio 17 sprites tile.

<img mk_tile_gms_corner/>

### GMS Corner + Side

Tile set for GameMaker studio 55 sprites tile.

<img mk_tile_gms_corner_side/>

### Godot Blob

Tile set for Godot tile.

<img mk_tile_godot/>

## Cropping

The <junc crop/> property control the trimming of the tile layer in each direction. Note that the crop only 
affect tile layer.

<img mk_tile_cropping/>

## Edge

Edge layer is used to draw sprite on the edge of the tile. Note that edge 

### <junc edge type/>

Edge type control whether to use a single surface for all edges or individual edge sprites.

When using <span class="inline-code">Uniform</span> type, the edge should be aligned to the top edge with 
the same dimension as the tile. The other edges will be generated by flipping or rotating the edge texture 
based on the <junc edge transform/> value.

<img mk_tile_edge_uniform/>

When using <span class="inline-code">individual</span> type, each edge should be aligned to the side it 
connected to.

### <junc edge sprite/>

Edge sprite property allows you to set the corner edge differently from the side.

<img mk_tile_edge_sprite/>

### <junc edge shift/>

Shift the edge of each side.

### <junc full edge/>

Whether to crop the edge of use the entire sprite.

<img mk_tile_edge_full/>

## Output

<junc output type/> control whether to output as a single spritesheet or sprite array.
